# 📖 User Guide

Comprehensive guide to using OmniBlocks features and capabilities.

## 🎯 Interface Overview

### Main Editor Components

1. **Stage**: The area where your project runs (600×480 pixels by default)
2. **Sprite Pane**: Shows all sprites and backdrops in your project
3. **Blocks Palette**: Categorized blocks for programming
4. **Code Area**: Where you assemble blocks into scripts
5. **Toolbar**: Quick access to common functions
6. **Status Bar**: Shows project information and status

### Navigation

- **Tabs**: Switch between different sprites and backdrops
- **Zoom**: Use Ctrl/Cmd + mouse wheel to zoom in/out
- **Fullscreen**: Press F11 or click the fullscreen button
- **Search**: Press Ctrl/Cmd + F to search blocks

## 🧩 Core Features

### Project Management

#### Creating Projects
- **New Project**: File → New (Ctrl/Cmd + N)
- **Templates**: Choose from various project templates
- **Remix**: Start from existing projects

#### Saving and Loading
- **Save**: File → Save (Ctrl/Cmd + S)
- **Save As**: File → Save As
- **Auto-Save**: Enabled by default (configurable in settings)
- **Version History**: Access previous versions

#### File Formats
- **SB3**: Standard Scratch 3.0 format (fully compatible)
- **SB2**: Scratch 2.0 format (import only)
- **OmniBlocks Project**: Enhanced format with additional metadata

### Sprite Management

#### Creating Sprites
- **Paint Editor**: Draw custom sprites
- **Upload**: Import image files (PNG, JPG, SVG, GIF)
- **Library**: Choose from built-in sprite library
- **Camera**: Capture from webcam
- **Surprise**: Random sprite selection

#### Sprite Properties
- **Name**: Customizable sprite names
- **Position**: X/Y coordinates on stage
- **Size**: Scale percentage
- **Direction**: Rotation angle
- **Visibility**: Show/hide toggle
- **Layering**: Front/back ordering

### Blocks and Scripting

#### Block Categories
- **Motion**: Movement and position controls
- **Looks**: Visual appearance and effects
- **Sound**: Audio playback and effects
- **Events**: Trigger-based programming
- **Control**: Flow control (loops, conditionals)
- **Sensing**: Input and detection
- **Operators**: Mathematical and logical operations
- **Variables**: Data storage and manipulation
- **My Blocks**: Custom blocks
- **Extensions**: Additional functionality

#### Advanced Scripting
- **Custom Blocks**: Create reusable code blocks
- **Cloning**: Create sprite copies at runtime
- **Broadcasting**: Message-based communication
- **Lists**: Advanced data structures
- **Procedures**: Function-like constructs

## 🚧 Troubleshooting Common Issues

### Performance Problems

- **Slow Execution**: Enable Turbo Mode, reduce sprite count
- **Memory Issues**: Close unused tabs, reduce project complexity
- **Rendering Lag**: Lower visual quality settings, reduce effects

### Compatibility Issues

- **Browser Problems**: Use Chrome or Edge for best compatibility
- **Device Limitations**: Check system requirements
- **Extension Conflicts**: Disable problematic extensions

### Project Issues

- **Corrupted Projects**: Use auto-recovery features
- **Missing Assets**: Check file references and paths
- **Script Errors**: Use debugging tools to identify issues
